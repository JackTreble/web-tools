# Implementation Plan: Free QR Code Generator with Custom Styling

**Branch**: `004-qr-code-generator` | **Date**: 2026-03-26 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/004-qr-code-generator/spec.md`

---

## Summary

Build a fully browser-local QR code generator that delivers custom colors, multiple output formats (PNG + SVG), and structured content templates (WiFi, vCard, SMS) — all features that competing tools lock behind paywalls. QR encoding is handled by a vendored JS library (~15 KB); the main implementation work is Canvas-based color rendering, SVG export, and the structured template forms.

---

## Technical Context

**Language/Version**: HTML5, CSS3, modern JavaScript (ES Modules, no build step)
**Primary Dependencies**: `qrcode` npm package (vendored as a browser-compatible UMD bundle under `/tools/vendor/qrcode/`) — handles QR matrix generation
**Storage**: N/A — all state is in-memory for the session
**Testing**: Manual browser smoke tests per acceptance scenarios
**Target Platform**: Static GitHub Pages; Chrome 90+, Firefox 88+, Safari 15+
**Project Type**: Single-page client-side web tool
**Performance Goals**: QR preview renders within 200 ms of any input change
**Constraints**: Zero server requests; offline-capable after first load; no watermark on exports; GitHub Pages compatible (no custom headers or server config)
**Scale/Scope**: Single HTML page tool; ~300–500 lines of JS total

## Phase Design

### Phase 1 — Setup

Scaffold the file structure and wire the HTML shell. No logic yet.

- Create `tools/qr-code-generator.html` from `tools/template.html` with correct title, meta description, and `<link>` / `<script>` tags pointing at `common.css`, `tools/qr-code-generator/style.css`, the vendored `qrcode.min.js`, and `tools/qr-code-generator/main.js`.
- Create `tools/qr-code-generator/style.css` stub.
- Create `tools/qr-code-generator/main.js` stub (ES Module).
- Vendor the `qrcode` library: use npm dev workflow to pin the package, copy the UMD browser bundle to `tools/vendor/qrcode/qrcode.min.js`, and commit it. Reference via a `<script>` tag — not via `node_modules`.

### Phase 2 — Foundational

Core in-memory state and QR rendering engine that all user stories depend on.

- **QR state object**: `{ payload: string, fgColor: string, bgColor: string, ecLevel: string, sizePx: number, mode: 'plain' | 'wifi' | 'vcard' | 'sms' }` — single source of truth.
- **`buildPayload(state)`**: Assembles the final QR string from state. For `plain` mode returns `state.payload` directly. For structured modes constructs the appropriate format string.
- **`renderQR(state)`**: Calls the vendored `qrcode` library with the current payload and settings, renders to an offscreen canvas with custom fg/bg colors applied, then copies to the visible preview canvas. Also triggers the SVG element update if SVG mode is enabled.
- **Debounced update loop**: Any change to any control calls `setState(patch)` → debounced 100 ms → `renderQR()`. Ensures live preview without thrashing on every keypress.
- **Contrast checker**: Given fg and bg hex values, computes relative luminance per WCAG 2.1 and returns a boolean `isLowContrast`.
- **Capacity checker**: After QR generation, reads the `qrcode` library's capacity metadata to detect if the payload exceeded limits and flags an overflow state.

### Phase 3 — User Story 1: Generate from URL/Text + Download (P1 MVP)

Build the minimal working tool: text input → live QR preview → PNG/SVG download.

- Plain-text/URL input field with real-time event listener.
- Preview canvas element sized to current `state.sizePx`.
- "Download PNG" button: calls `canvas.toBlob('image/png')`, wraps in `URL.createObjectURL`, triggers `<a>` download, then revokes the object URL.
- "Download SVG" button: serialises the QR SVG element (or constructs SVG from matrix) via `XMLSerializer`, wraps in a `Blob('image/svg+xml')`, triggers download.
- Clear/empty state: when input is empty show a placeholder message instead of a blank canvas.

### Phase 4 — User Story 2: Customize Appearance (P2)

Add styling controls that sit alongside the live preview.

- Foreground color: `<input type="color">` + hex text input, synced bidirectionally.
- Background color: same pattern.
- Error correction selector: `<select>` with Low / Medium / Quartile / High options and plain-language labels.
- Size slider: range input 200–2000 px with numeric readout.
- "Reset to Defaults" button: sets state to `{ fgColor: '#000000', bgColor: '#ffffff', ecLevel: 'M', sizePx: 512 }` and re-renders.
- Low-contrast warning banner: shown/hidden based on `isLowContrast()` result after each render.

### Phase 5 — User Story 3: Structured Content Types (P3)

Add template switcher and structured forms.

- Mode switcher: tab or `<select>` for Plain / WiFi / vCard / SMS.
- **WiFi form**: SSID (text), Password (text), Security (`<select>`: WPA/WEP/None). Assembles `WIFI:T:{sec};S:{ssid};P:{pwd};;` with proper escaping of special characters (`;`, `,`, `\`, `"`).
- **vCard form**: Name (text), Phone (text), Email (text). Assembles vCard 3.0 string.
- **SMS form**: Phone number (text), Message (textarea). Assembles `smsto:{phone}:{message}`.
- Capacity overflow warning: shown when payload for the chosen template exceeds QR limits.
- Switching modes resets the payload and re-renders an empty QR placeholder.

### Phase N — Polish & Cross-Cutting

- Mobile-responsive layout (tool usable on a phone screen).
- SEO: descriptive `<title>`, `<meta name="description">`, meaningful headings, "How to use" section in HTML.
- Cross-browser validation: test in Chrome, Firefox, Safari.
- Accessibility: labels on all inputs, sufficient color contrast in UI chrome, keyboard-navigable controls.
- Privacy validation: run DevTools Network audit during a full session; confirm zero outbound requests.

## Key Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| QR library | `qrcode` (npm, vendored UMD) | Smallest viable library with PNG canvas and SVG output modes; easily vendored; MIT licensed |
| Color rendering | Canvas paint pass | Draw QR pixels manually with `fillStyle` set to fg/bg colors; gives full color control without depending on library color support |
| SVG export | Library SVG output or `XMLSerializer` | If `qrcode` library provides SVG output, use it directly; otherwise construct from matrix. Evaluate at implementation time. |
| State management | Plain JS object + re-render | No framework needed; tool is simple enough that a single `state` object and a `renderQR()` function covers all interactions |
| Structured templates | In-JS string builders | Trivial to implement; no external dependency needed |
